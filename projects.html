Office Announcements Board

The Office Announcements board, codenamed Denko-Board, was a project that I worked on with a coworker during my time at Maverick Software Consulting. The Purpose of the project was to provide a kiosk-style webpage that would display news, weather, and music feeds for the office, as well as announcements that could be set by the manager.

My Role: I was a secondary developer on the project, and my main role was to maintain the server and add the weather functionality to the frontend. I also set up the Raspberry Pi server to run autonomously so that future users could simply plug it in to the power and the internet and it would automatically start serving.

Group Members: 2

Technologies: a Node.js server with an Angular/ Bootstrap frontend that ran autonomously on a Raspberry Pi running Raspbian (a debian distro)

Experience gained: This was my first exposure to web development, including Javascript and the entire process of running a server that serves webpages. I also learned a great deal about linux, including installation, setup, basic commands, and cron.

QWOP AI

The QWOP AI, or "Forrest Grump," is a machine learning project that aimed to make a program that could learn how to play the flash game QWOP via interactions and data from the game played in a web browser. The project ended with moderate success, as most of the time the runner avatar would "learn" how to slowly advance.

My Role: I was tasked with implementing the Q-learning model, parsing information from the game via taking images and parsing out various bits of data, and driving interaction with the game via Selenium Webdriver.

Group Members: 4

Technologies: Java, using Selenium Webdriver to interact with the game through Chrome browser.

Experience gained: I learned a huge amount about machine learning and specifically the Q-learning model. I also learned some more advanced techniques with Selenium Webdriver, and I wrote some rudimentary image processing and practiced optimization of algorithms.

Better BlackBoard

Better BlackBoard is a project that aims to "do right" all of the things that are frustrating or broken on the ISU Blackboard page and application, such as UI responsiveness, Browser support, various UI bugs or poor design, and extra features aimed at incorporating other services that students have to use for certain classes, such as Piazza and TopHat.

My Role: I am the sole developer on this project, which has gone through a few iterations.

Group Members: 1

Technologies: Node.js server with a (temporary) Firebase DB, attached to a UI that runs on React for the web and React Native for Android and iOS mobile devices. This React frontend also uses a Flux style dataflow pattern, specifically redux.

Experience gained: This project has been instrumental for me in learning and keeping up with new versions of Javascript and frontend frameworks. It is also a good way of applying the things I learn at my Internship to a different project so I can expand on what I know even more.
